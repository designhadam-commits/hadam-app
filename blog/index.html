🔧 n8n 설정 가이드

① 키워드 검색량 프록시  |  ② 블로그 자동 발행
A
🔍 키워드 검색량 프록시 워크플로우
앱 → n8n → 네이버 검색광고 API 순서로 중계해서 CORS 없이 검색량을 가져옵니다.
① Webhook 노드
// Webhook 노드 설정 HTTP Method: POST Path: kw-volume Response Mode: Last Node // 이 URL을 앱의 n8n URL 칸에 입력하세요 https://your-n8n.com/webhook/kw-volume
② Code 노드 — 네이버 API 서명 + 호출
// Code 노드 (JavaScript) const crypto = require('crypto'); const API_KEY = 'YOUR_ACCESS_LICENSE'; const SECRET_KEY = 'YOUR_SECRET_KEY'; const CUSTOMER_ID = 'YOUR_CUSTOMER_ID'; const keywords = $input.first().json.body.keywords; const timestamp = Date.now().toString(); const signature = crypto .createHmac('sha256', SECRET_KEY) .update(`${timestamp}.${API_KEY}`) .digest('base64'); const params = keywords .map(k => `hintKeywords=${encodeURIComponent(k)}`) .join('&'); const res = await fetch( `https://api.naver.com/keywordstool?${params}&showDetail=1`, { headers: { 'X-Timestamp': timestamp, 'X-API-KEY': API_KEY, 'X-Customer': CUSTOMER_ID, 'X-Signature': signature } } ); const data = await res.json(); // 앱이 기대하는 형식으로 가공 const results = {}; (data.keywordList || []).forEach(item => { results[item.relKeyword] = { pc: item.monthlyPcQcCnt || 0, mobile: item.monthlyMobileQcCnt || 0, total: (item.monthlyPcQcCnt || 0) + (item.monthlyMobileQcCnt || 0) }; }); return [{ json: { results } }];
💡 사용 방법
1. n8n에서 위 워크플로우 생성 후 Webhook URL 복사
2. 앱 왼쪽 🔑 검색량 조회 연동 카드에 URL 붙여넣기
3. 💾 저장 → 🔍 연결 테스트 클릭
4. 이제 "AI 연관 키워드 추천" 버튼을 누르면 검색량이 자동으로 표시돼요!

📅 블로그 자동 발행 워크플로우
매일 09:00 자동 실행 → Google Sheets에서 주제/키워드 → Claude API 글 생성 → 검증 → 발행
1
Google Sheets — 콘텐츠 계획표 준비
아래 컬럼으로 시트를 만들어 두면 날짜별로 자동으로 다른 글이 생성돼요.
// Google Sheets 컬럼 구조 날짜 | 화자 | 주제 | 메인키워드 | 연관키워드(쉼표구분) | 발행여부 // 예시 행 2026-06-02 | customer | 부산 롯데캐슬 27평 올수리 | 부산아파트인테리어,27평인테리어 | 부산롯데캐슬,올수리비용,구축아파트리모델링,부산인테리어업체 | N 2026-06-03 | manager | 인테리어 공정 순서 완벽 정리 | 인테리어공정,부산아파트인테리어 | 부산리모델링,공정관리,목공전기타일,하자예방 | N
2
Schedule Trigger + Google Sheets Read
매일 09:00 (Asia/Seoul) 실행 → Google Sheets: Get Row로 오늘 날짜 행 조회
3
Set 노드 — 화자별 프롬프트 분기
// IF 노드: voice === "customer" → 고객후기 프롬프트 // voice === "manager" → 실장글 프롬프트 // 고객후기 프롬프트 핵심 지시 CUSTOMER_STYLE: "당신은 디자인하담에서 시공을 마친 만족한 고객입니다. 도입부는 반드시 '안녕하세요! 오늘은 드디어~'로 시작. Before→After 공간별 구조. ~어요/~죠 구어체. 정말/너무나/완전히/대성공 등 감탄 표현 자연스럽게 사용. 마무리: 디자인하담 강력 추천 CTA" // 실장글 프롬프트 핵심 지시 MANAGER_STYLE: "당신은 디자인하담 실장입니다. 고객 질문이나 현장 경험으로 시작. 결론 먼저 → 원인 → 해결책 → 결론 구조. 현장에서/실제로/경험상 전문가 표현. ~입니다/~합니다 격식체 + 친근함. 마무리: 디자인하담 상담 문의 CTA"
4
Code 노드 — 글자 수 & 키워드 검증
const post = JSON.parse($json.content[0].text); const full = [post.intro, ...post.sections.map(s=>s.content), post.cta].join(' '); const noSpace = full.replace(/\s/g,'').length; // 공백 제외 const allKws = [...$json.main_kws, ...$json.rel_kws]; const counts = {}; allKws.forEach(kw => { counts[kw] = (full.match(new RegExp(kw, 'g')) || []).length; }); const kwPass = Object.values(counts).every(c => c >= $json.min_count); const charPass = noSpace >= 1500; return [{json: { ...post, noSpace, counts, kwPass, charPass, pass: kwPass && charPass }}];
5
IF 분기 → 발행 or 재생성 (최대 3회)
pass === true → 발행 (Gmail 초안 or Notion 저장 or 네이버 API)
pass === false → HTTP Request 재생성, 3회 초과 시 Slack 알림
💡 화자 교차 운영 팁
월·수·금 — 고객 후기 (검색 신뢰도 ↑)
화·목 — 실장 정보성 글 (전문성 & SEO ↑)
Google Sheets에 요일별로 화자를 미리 설정해두면 완전 자동으로 교차 발행됩니다.

⚙️ Make 자동화 설정

Schedule → Google Sheets → Router(화자분기) → Claude → 검증 → 발행
1
Schedule + Google Sheets
Make의 Google Sheets: Search Rows로 오늘 날짜 행 조회. 동일한 시트 구조 사용.
2
Router — 화자별 분기
Make의 Router 모듈로 voice === "customer" / voice === "manager" 두 경로로 분기. 각 경로에서 다른 프롬프트로 Claude API 호출.
3
Filter → 발행
공백제외 1,500자 이상 & 키워드 5회 이상 Filter 통과 시 발행. 미달 시 재시도 경로로 분기.
